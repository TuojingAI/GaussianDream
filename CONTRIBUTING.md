# Contributing to GaussianDream

We welcome contributions, improvements, and bug reports. Please open an issue or pull request if you find something that should be fixed or documented better.

## Issues and feature requests

Before filing an issue, please check whether it has already been reported. When you open an issue, include:

- your OS type and version
- your Python version
- steps to reproduce
- any traceback or log output
- any other detail that helps reproduce the problem

If you want to request a feature, please include:

- the motivation
- the problem you are trying to solve
- the intended use case
- any constraints we should know about

## Pull requests

Please make PRs as small and focused as possible. For code changes, run the relevant tests and formatting checks before submitting.

If your change touches benchmark or evaluation code, please verify the LIBERO and RoboCasa examples still work end to end.
